# Controller

> **Category:** GRASP (General Responsibility Assignment Software Patterns)  
> **Type:** Responsibility Assignment Pattern  
> **Difficulty:** ⭐⭐ Intermediate

---

## 📖 Definition

**Controller** is a principle for assigning the responsibility of handling system events and coordinating system operations. It states:

> **Assign the responsibility for handling a system event to a class that represents one of these choices:**
> 1. **Represents the overall system, device, or subsystem** (facade controller)
> 2. **Represents a use case scenario** within which the system event occurs (use case controller)

In simpler terms: **The controller is the first object beyond the UI layer that receives and coordinates system operations.**

---

## 🔍 Explanation

The Controller pattern answers the question: **"What first object beyond the UI layer receives and coordinates ('controls') a system operation?"**

### Core Principle
- **Separation of Concerns**: Separate UI from business logic
- **Coordination**: Controller coordinates the work, doesn't do it all
- **Single Entry Point**: Centralized handling of system events
- **Delegation**: Controller delegates to domain objects

### Key Concepts

**System Event**: A message generated by an external actor (user, external system, timer)
- Examples: "User clicks Submit", "Payment received", "Order placed"

**System Operation**: A high-level operation the system provides
- Examples: `placeOrder()`, `processPayment()`, `registerUser()`

### Two Types of Controllers

#### 1. Facade Controller (System/Subsystem Controller)
Represents the entire system or a subsystem
- Examples: `System`, `Application`, `PaymentSystem`
- Use when: Few system operations, simple coordination

#### 2. Use Case Controller (Session Controller)
Represents a specific use case or user session
- Examples: `PlaceOrderHandler`, `LoginSession`, `CheckoutProcess`
- Use when: Many operations, complex workflows, multiple steps

---

## 🎯 Problem It Solves

### Problems Without Controller Pattern:
1. **Business Logic in UI**: Domain logic mixed with presentation
2. **Low Reusability**: Cannot reuse logic from different UIs (web, mobile, API)
3. **Difficult Testing**: Cannot test business logic without UI
4. **Poor Maintainability**: Changes ripple through UI and logic
5. **Bloated UI Components**: UI classes become God objects

### Example of Bad Design:
```python
# ❌ BAD: Business logic in UI/View
class OrderView:
```

## ✅ Solution With Controller Pattern

```python
# ✅ GOOD: Controller separates concerns

# UI Layer - only handles presentation
class OrderView:
    def __init__(self):
        self.controller = OrderController()
    
    def on_submit_button_click(self):
        """UI only gathers input and delegates"""
        product_id = self.product_dropdown.get_value()
        quantity = self.quantity_field.get_value()
        
        try:
            # Delegate to controller
            order = self.controller.place_order(product_id, quantity)
            self.show_success(f"Order #{order.id} placed successfully!")
        except InsufficientStockError as e:
            self.show_error(str(e))
        except Exception as e:
            self.show_error("Failed to place order")


# Controller Layer - coordinates system operations
class OrderController:
    def __init__(self):
        self.order_repository = OrderRepository()
        self.product_repository = ProductRepository()
    
    def place_order(self, product_id: int, quantity: int):
        """System operation handler - coordinates work"""
        # 1. Get domain objects (delegates to repositories)
        product = self.product_repository.find_by_id(product_id)
        if not product:
            raise ProductNotFoundError()
        
        # 2. Delegate business logic to domain objects
        if not product.has_sufficient_stock(quantity):
            raise InsufficientStockError(f"Only {product.stock} available")
        
        # 3. Create and configure domain objects
        order = Order()
        order.add_item(product, quantity)
        
        # 4. Delegate to domain for calculations
        order.calculate_total()
        
        # 5. Update domain state
        product.reduce_stock(quantity)
        
        # 6. Delegate persistence
        self.order_repository.save(order)
        self.product_repository.save(product)
        
        return order


# Domain Layer - encapsulates business logic
class Product:
    def __init__(self, id, name, price, stock):
        self.id = id
        self.name = name
        self.price = price
        self.stock = stock
    
    def has_sufficient_stock(self, quantity):
        """Domain logic in domain object"""
        return self.stock >= quantity
    
    def reduce_stock(self, quantity):
        if not self.has_sufficient_stock(quantity):
            raise InsufficientStockError()
        self.stock -= quantity


class Order:
    def __init__(self):
        self.items = []
        self.total = 0
    
    def add_item(self, product, quantity):
        item = OrderItem(product, quantity)
        self.items.append(item)
    
    def calculate_total(self):
        """Domain calculates its own total"""
        self.total = sum(item.get_subtotal() for item in self.items)
```

### Example 1: User Authentication System

```python
from datetime import datetime, timedelta
import hashlib
from typing import Optional

# Use Case Controller - represents authentication workflow
class AuthenticationController:
    """Controller for user authentication use case"""
    
    def __init__(self, user_repository, session_repository):
        self.user_repository = user_repository
        self.session_repository = session_repository
    
    def login(self, username: str, password: str) -> 'UserSession':
        """System operation: handle login request"""
        # 1. Find user (delegate to repository)
        user = self.user_repository.find_by_username(username)
        
        if not user:
            raise AuthenticationError("Invalid credentials")
        
        # 2. Validate credentials (delegate to domain)
        if not user.verify_password(password):
            user.record_failed_login()
            self.user_repository.save(user)
            raise AuthenticationError("Invalid credentials")
        
        # 3. Check account status (domain logic)
        if user.is_locked():
            raise AccountLockedError("Account locked due to multiple failed attempts")
        
        # 4. Create session (domain creation)
        session = user.create_session()
        
        # 5. Update state
        user.record_successful_login()
        
        # 6. Persist (delegate to repository)
        self.session_repository.save(session)
        self.user_repository.save(user)
        
        return session
    
    def logout(self, session_token: str):
        """System operation: handle logout"""
        session = self.session_repository.find_by_token(session_token)
        
        if session:
            session.terminate()
            self.session_repository.save(session)
    
    def validate_session(self, session_token: str) -> bool:
        """System operation: check if session is valid"""
        session = self.session_repository.find_by_token(session_token)
        
        if not session:
            return False
        
        if session.is_expired():
            session.terminate()
            self.session_repository.save(session)
            return False
        
        # Extend session on activity
        session.extend()
        self.session_repository.save(session)
        return True


# Domain Objects
class User:
    def __init__(self, username, password_hash):
        self.username = username
        self.password_hash = password_hash
        self.failed_login_attempts = 0
        self.last_login = None
        self.is_active = True
    
    def verify_password(self, password: str) -> bool:
        """Domain logic: password verification"""
        hashed = hashlib.sha256(password.encode()).hexdigest()
        return hashed == self.password_hash
    
    def is_locked(self) -> bool:
        """Domain logic: account lock check"""
        return self.failed_login_attempts >= 5
    
    def record_failed_login(self):
        self.failed_login_attempts += 1
    
    def record_successful_login(self):
        self.failed_login_attempts = 0
        self.last_login = datetime.now()
    
    def create_session(self) -> 'UserSession':
        """User creates its session (Creator pattern)"""
        return UserSession(self)


class UserSession:
    def __init__(self, user: User):
        self.user = user
        self.token = self._generate_token()
        self.created_at = datetime.now()
        self.expires_at = datetime.now() + timedelta(hours=24)
        self.is_active = True
    
    def _generate_token(self) -> str:
        import secrets
        return secrets.token_urlsafe(32)
    
    def is_expired(self) -> bool:
        return datetime.now() > self.expires_at
    
    def terminate(self):
        self.is_active = False
    
    def extend(self):
        """Extend session on activity"""
        self.expires_at = datetime.now() + timedelta(hours=24)


# UI Layer
class LoginView:
    def __init__(self):
        self.controller = AuthenticationController(
            UserRepository(),
            SessionRepository()
        )
    
    def on_login_click(self):
        """UI only gathers input and displays results"""
        username = self.username_input.get_text()
        password = self.password_input.get_text()
        
        try:
            session = self.controller.login(username, password)
            self.store_session_token(session.token)
            self.navigate_to_dashboard()
        except AccountLockedError as e:
            self.show_error(str(e))
        except AuthenticationError as e:
            self.show_error(str(e))
```

### Example 2: E-commerce Checkout Process

```python
from decimal import Decimal
from typing import List

# Use Case Controller - represents checkout workflow
class CheckoutController:
    """Controller for checkout use case - multi-step process"""
    
    def __init__(self, cart_repo, order_repo, payment_gateway, inventory_service):
        self.cart_repo = cart_repo
        self.order_repo = order_repo
        self.payment_gateway = payment_gateway
        self.inventory_service = inventory_service
    
    def initiate_checkout(self, user_id: int):
        """Step 1: Start checkout process"""
        cart = self.cart_repo.find_by_user(user_id)
        
        if cart.is_empty():
            raise EmptyCartError("Cannot checkout with empty cart")
        
        # Validate inventory availability
        for item in cart.items:
            if not self.inventory_service.is_available(item.product_id, item.quantity):
                raise InsufficientInventoryError(f"Product {item.product_id} not available")
        
        # Reserve inventory
        for item in cart.items:
            self.inventory_service.reserve(item.product_id, item.quantity)
        
        return cart
    
    def apply_discount_code(self, cart_id: int, code: str):
        """Step 2: Apply discount"""
        cart = self.cart_repo.find_by_id(cart_id)
        discount = self._validate_discount_code(code)
        
        # Domain object applies discount
        cart.apply_discount(discount)
        self.cart_repo.save(cart)
        
        return cart
    
    def set_shipping_address(self, cart_id: int, address_data: dict):
        """Step 3: Set shipping address"""
        cart = self.cart_repo.find_by_id(cart_id)
        
        # Create address (domain object)
        address = Address.from_dict(address_data)
        cart.set_shipping_address(address)
        
        # Calculate shipping (delegate to shipping service)
        shipping_cost = self._calculate_shipping(cart, address)
        cart.set_shipping_cost(shipping_cost)
        
        self.cart_repo.save(cart)
        return cart
    
    def process_payment(self, cart_id: int, payment_details: dict):
        """Step 4: Process payment and create order"""
        cart = self.cart_repo.find_by_id(cart_id)
        
        # Validate cart is ready
        if not cart.is_ready_for_payment():
            raise InvalidCheckoutStateError("Cart not ready for payment")
        
        # Create order from cart (domain logic)
        order = cart.create_order()
        
        # Process payment (delegate to payment gateway)
        try:
            payment = self.payment_gateway.charge(
                amount=order.get_total(),
                payment_details=payment_details
            )
            
            order.mark_as_paid(payment)
            
            # Confirm inventory reservation
            for item in order.items:
                self.inventory_service.confirm_reservation(item.product_id)
            
            # Save order
            self.order_repo.save(order)
            
            # Clear cart
            cart.clear()
            self.cart_repo.save(cart)
            
            return order
            
        except PaymentFailedError as e:
            # Release inventory reservation
            for item in cart.items:
                self.inventory_service.release_reservation(item.product_id)
            raise
    
    def _validate_discount_code(self, code: str):
        # Simplified - would normally query discount repository
        pass
    
    def _calculate_shipping(self, cart, address):
        # Simplified - would use shipping service
        return Decimal("9.99")


# Domain Objects
class ShoppingCart:
    def __init__(self, user_id):
        self.user_id = user_id
        self.items = []
        self.discount = None
        self.shipping_address = None
        self.shipping_cost = Decimal("0")
    
    def is_empty(self) -> bool:
        return len(self.items) == 0
    
    def apply_discount(self, discount):
        self.discount = discount
    
    def set_shipping_address(self, address):
        self.shipping_address = address
    
    def set_shipping_cost(self, cost):
        self.shipping_cost = cost
    
    def is_ready_for_payment(self) -> bool:
        return (not self.is_empty() and 
                self.shipping_address is not None)
    
    def create_order(self):
        """Cart creates Order (Creator pattern)"""
        order = Order(self.user_id)
        
        for item in self.items:
            order.add_item(item.product_id, item.quantity, item.price)
        
        if self.discount:
            order.apply_discount(self.discount)
        
        order.set_shipping_address(self.shipping_address)
        order.add_shipping_cost(self.shipping_cost)
        
        return order
    
    def clear(self):
        self.items.clear()
        self.discount = None
        self.shipping_address = None


class Order:
    def __init__(self, user_id):
        self.user_id = user_id
        self.items = []
        self.discount = None
        self.shipping_address = None
        self.shipping_cost = Decimal("0")
        self.status = "pending"
        self.payment = None
    
    def add_item(self, product_id, quantity, price):
        self.items.append(OrderItem(product_id, quantity, price))
    
    def apply_discount(self, discount):
        self.discount = discount
    
    def set_shipping_address(self, address):
        self.shipping_address = address
    
    def add_shipping_cost(self, cost):
        self.shipping_cost = cost
    
    def get_subtotal(self) -> Decimal:
        return sum(item.get_total() for item in self.items)
    
    def get_total(self) -> Decimal:
        subtotal = self.get_subtotal()
        
        if self.discount:
            subtotal = self.discount.apply(subtotal)
        
        return subtotal + self.shipping_cost
    
    def mark_as_paid(self, payment):
        self.payment = payment
        self.status = "paid"
```

### Example 3: Facade Controller for Simple System

```python
# Facade Controller - represents entire system
class LibrarySystem:
    """Simple facade controller for library operations"""
    
    def __init__(self):
        self.book_repository = BookRepository()
        self.member_repository = MemberRepository()
        self.loan_repository = LoanRepository()
    
    def register_member(self, name: str, email: str):
        """System operation: register new member"""
        # Check if email already exists
        existing = self.member_repository.find_by_email(email)
        if existing:
            raise DuplicateMemberError("Email already registered")
        
        # Create member (domain object)
        member = Member(name, email)
        
        # Persist
        self.member_repository.save(member)
        return member
    
    def add_book(self, title: str, author: str, isbn: str):
        """System operation: add book to library"""
        # Check for duplicates
        existing = self.book_repository.find_by_isbn(isbn)
        if existing:
            raise DuplicateBookError("Book already exists")
        
        # Create book
        book = Book(title, author, isbn)
        
        # Persist
        self.book_repository.save(book)
        return book
    
    def borrow_book(self, member_id: int, book_isbn: str):
        """System operation: member borrows book"""
        # Get domain objects
        member = self.member_repository.find_by_id(member_id)
        book = self.book_repository.find_by_isbn(book_isbn)
        
        if not member:
            raise MemberNotFoundError()
        if not book:
            raise BookNotFoundError()
        
        # Validate (domain logic)
        if not book.is_available():
            raise BookNotAvailableError()
        
        if member.has_overdue_books():
            raise OverdueBooksError("Cannot borrow with overdue books")
        
        if member.has_reached_loan_limit():
            raise LoanLimitError("Maximum loans reached")
        
        # Create loan (domain operation)
        loan = member.borrow_book(book)
        
        # Update book status
        book.mark_as_borrowed()
        
        # Persist changes
        self.loan_repository.save(loan)
        self.book_repository.save(book)
        self.member_repository.save(member)
        
        return loan
    
    def return_book(self, loan_id: int):
        """System operation: return borrowed book"""
        loan = self.loan_repository.find_by_id(loan_id)
        
        if not loan:
            raise LoanNotFoundError()
        
        if loan.is_returned:
            raise AlreadyReturnedError()
        
        # Calculate late fee if overdue
        late_fee = loan.calculate_late_fee()
        
        # Return book (domain logic)
        loan.return_book()
        book = loan.book
        book.mark_as_available()
        
        # Persist
        self.loan_repository.save(loan)
        self.book_repository.save(book)
        
        return late_fee
```

## ✅ Pros and Cons

### Pros ✅

1. **Separation of Concerns**
   - UI separated from business logic
   - Clear layer boundaries

2. **Reusability**
   - Same controller from web, mobile, API, CLI
   - Business logic not tied to presentation

3. **Testability**
   - Can test business logic without UI
   - Mock controllers easily

4. **Maintainability**
   - Changes localized to appropriate layers
   - Easier to understand responsibilities

5. **Flexibility**
   - Change UI without touching business logic
   - Change business logic without touching UI

6. **Single Entry Point**
   - Clear place for system operations
   - Easier to add cross-cutting concerns (logging, security)

### Cons ❌

1. **Bloated Controllers (God Object)**
   - Controllers can become too large
   - Tendency to put all logic in controller

2. **Extra Layer**
   - Adds complexity for very simple operations
   - Overkill for trivial CRUD

3. **Over-Coordination**
   - Controllers doing work instead of delegating
   - Anemic domain objects result

4. **Confusion with MVC**
   - Term "Controller" overloaded
   - Not the same as MVC Controller exactly

5. **Stateless Nature**
   - Controllers typically stateless
   - Multi-step workflows need session management

---

## 💡 When to Use

### ✅ Use Controller When:

1. **Separating UI from Logic**
   - Need to support multiple UIs (web, mobile, API)
   - Example: REST API and web interface for same system

2. **System-Level Operations**
   - Operations that involve multiple domain objects
   - Example: Checkout process involving cart, payment, inventory

3. **Use Case Workflows**
   - Multi-step processes
   - Example: User registration, order processing

4. **Testing Requirements**
   - Need to test logic without UI
   - Example: Automated testing of business rules

5. **Coordination Needed**
   - Operations require coordinating several objects
   - Example: Transfer money between accounts

### ❌ Don't Use Controller When:

1. **Simple CRUD**
   - Direct database access sufficient
   - Example: Simple list/detail views

2. **Pure Domain Logic**
   - Operation entirely within one domain object
   - Example: Calculating order total

3. **Already Have Service Layer**
   - Don't duplicate responsibilities
   - Choose Controller OR Service, not both blindly

4. **Real-time Systems**
   - Need event-driven architecture
   - Use event handlers or message processors

## 🚫 Common Mistakes

### Mistake 1: Bloated Controller (God Object)
```python
# ❌ BAD: Controller doing too much
class OrderController:
    def place_order(self, product_id, quantity, user_id):
        # Controller doing business logic
        product = self.db.query("SELECT * FROM products WHERE id = ?", product_id)
        
        # Calculations in controller
        subtotal = product['price'] * quantity
        tax = subtotal * 0.1
        shipping = 9.99 if subtotal < 50 else 0
        total = subtotal + tax + shipping
        
        # Validation in controller
        if product['stock'] < quantity:
            raise InsufficientStockError()
        
        # Database operations in controller
        self.db.execute("INSERT INTO orders ...")
        self.db.execute("UPDATE products SET stock = ...")
        
        # Email logic in controller
        self.send_confirmation_email(user_id, total)

# ✅ GOOD: Controller coordinates, delegates work
class OrderController:
    def place_order(self, product_id, quantity, user_id):
        # Delegate to repositories
        product = self.product_repo.find(product_id)
        user = self.user_repo.find(user_id)
        
        # Delegate validation to domain
        if not product.has_sufficient_stock(quantity):
            raise InsufficientStockError()
        
        # Domain objects do their work
        order = user.create_order()
        order.add_item(product, quantity)
        order.calculate_total()  # Domain calculates
        
        # Coordinate persistence
        product.reduce_stock(quantity)
        self.order_repo.save(order)
        self.product_repo.save(product)
        
        # Delegate notification
        self.notification_service.send_order_confirmation(order)
        
        return order
```

### Mistake 2: Anemic Domain Model
```python
# ❌ BAD: All logic in controller, domain is just data
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email
        self.orders = []

class UserController:
    def place_order(self, user_id, product_id, quantity):
        user = self.user_repo.find(user_id)
        
        # Controller doing domain logic
        if len(user.orders) >= 10:
            raise TooManyOrdersError()
        
        order = Order()
        user.orders.append(order)
        # All logic here, domain just stores data

# ✅ GOOD: Domain has behavior
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email
        self._orders = []
        self.max_concurrent_orders = 10
    
    def can_place_order(self) -> bool:
        """Domain knows its rules"""
        active_orders = [o for o in self._orders if not o.is_completed]
        return len(active_orders) < self.max_concurrent_orders
    
    def place_order(self) -> Order:
        """Domain creates its orders"""
        if not self.can_place_order():
            raise TooManyOrdersError()
        
        order = Order(self)
        self._orders.append(order)
        return order

class UserController:
    def place_order(self, user_id, product_id, quantity):
        user = self.user_repo.find(user_id)
        
        # Just coordinate, let domain do its work
        order = user.place_order()
        product = self.product_repo.find(product_id)
        order.add_item(product, quantity)
        
        self.order_repo.save(order)
        return order
```

## 🔗 Related Patterns

### GRASP Patterns
- **Information Expert**: Controller delegates to experts
- **Creator**: Controller may use creators
- **Low Coupling**: Controller should not be tightly coupled to UI
- **High Cohesion**: Keep controller focused on coordination

### GoF Patterns
- **Facade**: Facade Controller is similar
- **Command**: Can encapsulate system operations
- **Strategy**: Different controllers for different strategies
- **Observer**: Controllers can observe domain events

### Architectural Patterns
- **Layered Architecture**: Controllers live in application layer
- **MVC**: Controller in MVC is similar but UI-focused
- **Hexagonal Architecture**: Controllers are "adapters"
- **Clean Architecture**: Controllers are interface adapters

## 📊 Controller vs Service Layer

| Aspect | Controller | Service Layer |
|--------|-----------|---------------|
| **Purpose** | Handle system events from UI | Business transaction scripts |
| **Scope** | Per use case or system | Per business capability |
| **Coupling** | May couple to UI frameworks | Independent of UI |
| **Granularity** | Coarse-grained operations | Can be fine or coarse |
| **State** | Typically stateless | Usually stateless |
| **Example** | `PlaceOrderController` | `OrderService` |

### When to Use Both?
```python
# UI Layer
class OrderView:
    def on_submit(self):
        self.controller.place_order(...)

# Controller Layer (coordinates HTTP/UI concerns)
class OrderController:
    def place_order(self, request_data):
        # Handle HTTP/UI concerns
        user_id = request_data.get_user_id()
        
        # Delegate to service
        order = self.order_service.place_order(user_id, ...)
        
        # Return UI-appropriate response
        return {'order_id': order.id}

# Service Layer (business logic)
class OrderService:
    def place_order(self, user_id, product_id, quantity):
        # Business transaction
        user = self.user_repo.find(user_id)
        order = user.place_order()
        # ... business logic
        return order

# Domain Layer
class User:
    def place_order(self):
        # Domain logic
        pass
```
